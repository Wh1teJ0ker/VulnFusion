## 系统目录设计

```undefined
vulautoscan/
├── cmd/                        # 启动入口（CLI/Web）
│   └── cli.go
├── internal/
│   ├── scanner/                # 调度器（统一调核、插件、任务队列）
│   ├── nuclei/                 # nuclei模板扫描模块
│   ├── plugin/                 # 支持 Python/Bash/Go 等脚本插件执行
│   ├── task/                   # 多线程任务分发管理
│   ├── result/                 # 扫描结果处理与导出
│   ├── config/                 # 配置管理（支持 YAML）
│   └── vulnfeed/               # 漏洞情报插件接口
│       ├── github.go          # 默认 GitHub 插件实现
│       ├── plugin.go          # FeedPlugin 接口定义
│       └── register.go        # 插件注册与调用机制
├── web/
│   ├── ui/                     # 前端静态文件（Vue 或 HTML）
│   └── server.go               # Gin 控制器，绑定路由
├── plugins/                   # 用户脚本插件存放目录
├── templates/                 # nuclei模板目录
├── targets.txt                # 目标资产列表
├── config.yaml                # 配置文件
├── main.go                    # 统一启动入口
└── go.mod

```
